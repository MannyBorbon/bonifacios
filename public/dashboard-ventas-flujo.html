3 · Qué hace cada paso (resumen)

Paso 1 SQL Server (SR)

Tablas cheques, chequespagos, detalle e importes. Es la fuente de verdad del POS.

Paso 2 Script en el PC (sync)

Lee SR, arma JSON por lotes y envía a sync.php. Corrige desglose efectivo/tarjeta cuando las líneas no mapean solo a 1/2/3.

Paso 3 sync.php (API)

Valida API key y hace upsert en sr_sales, ítems y sr_cheque_payments.

Paso 4 MySQL

Guarda tickets y montos que consume el dashboard. Índices en sale_datetime, status, sr_ticket_id aceleran consultas.

Paso 5 sales.php

sections=core: stats + periodo + ventas + métodos de pago sin analytics pesado. sections=heavy: solo analytics/meseros/top.

Paso 6 Sales.jsx

Pide core primero, fusiona heavy después. gen + abort evita mostrar “ayer” con datos de “hoy” si cambió el filtro.

Paso 7 caja / turnos

cash-movements.php y shifts.php en paralelo; no bloquean el primer pintado de ventas.

Extra SalesWidget

Solo range=today&sections=core; usa stats.today y stats.yesterday.
